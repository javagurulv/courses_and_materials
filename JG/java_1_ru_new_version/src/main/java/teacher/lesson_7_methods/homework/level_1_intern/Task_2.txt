Если присмотреться к коду класса FizzBuzz получше, то можно увидеть,
что проверки деления числа без остатка на 3 и на 5 встречаются в коде по два раза:
- (number % 3 == 0)
- (number % 5 == 0)

У кода этих проверок есть еще один большой недостаток с точки зрения
читаемости и быстрого понимания кода. Эти проверки написаны сугубо
техническим языком (с помощью оператора % в Java), если не знать, как
работает оператор % в Java, то понять эти условия становится практически
не возможным.

В программировании есть классный инструмент, который позволит нам
избавиться от дубликатов в коде и решит проблему читаемости и
понимания кода. Этот механизм называется методы!

Метод в программировании это кусочек кода написанный отдельно от
основной программы и решающий одну конкретную задачу. Метод в программировании
как функция в математике, ей на вход подаются конкретные значения
параметров, а она вычисляет какую то формулу и возвращает полученный
результат. В отличие от математики методы в программировании могут
решать практически любые задачи!

Например, в нашем примере мы можем создать метод, который бы определял
делиться ли число на 3 без остатка или нет.
