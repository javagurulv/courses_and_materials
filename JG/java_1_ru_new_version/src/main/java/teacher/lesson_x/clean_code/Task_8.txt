Цель: покрытие кода тестами.

В процессе создания качественного кода играют большую роль тесты
(unit, integration, acceptance, ...). Тестирование своего (и не только)
кода это неотъемлемая и очень важная часть работы программиста. Время,
которое будет потрачено на обучение тестированию, не будет потрачено
впустую.

Без качественных тестов невозможно создать качественное решение!

Создание качественного кода это в первую очередь процесс и
автоматизация тестирования занимает в нём одно из самых важных мест.

Одна из метрик используемых для анализа тестов это покрытие строк кода
тестами. Сколько строк вашего продакшн кода покрыто тестами (сколько
% строк продакшн кода покрыто тестами): 60%, 80%, 100%?

Задача 1.11: возьмите одно из ваших предыдущих решений и добейтесь
60% покрытия unit тестами.

Задача 1.12: возьмите другое ваше решение и добейтесь
80% покрытия unit тестами.

Задача 1.13: возьмите другое ваше решение и добейтесь
100% покрытия unit тестами.

PS: процент покрытия продакшн кода тестами собирает и показывает
JUnit библиотека при запуске тестов. Разберитесь (загуглите) как это
делается в вашей среде разработки.
