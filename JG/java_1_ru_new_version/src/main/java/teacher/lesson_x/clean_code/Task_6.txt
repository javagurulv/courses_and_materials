В (clean code) качественном коде порой уместны методы длиной в
ОДНУ строку кода! Самый простой пример это сложное длинное условие
в операторе if. Как сделать его простым для понимания? Вынесите
его в отдельный метод длиной всего в одну строку кода! Самое
удивительное это то, что вы сможете дать этому сложному условию
название с помощью названия метода. Название метода может сильно
облегчить понимание этого условия.

Вынося кусочек кода в отдельный метод вы не только структурируете
и делите свой код на кусочки, но и получаете возможность дать этим
кусочкам хорошие названия!

Хорошие названия методов как названия глав книги помогают лучше
ориентироваться в коде и быстрее его читать и понимать.
