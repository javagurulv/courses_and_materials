Цель: создание качественных тестов.

Процент покрытия продакшн кода unit тестами ничего не говорит о
качестве тестов, он показывает только их наличие.

К созданию качественного кода приводят качественные тесты.

Что такое качественные тесты? Качественный тест это тест, который
проверяет выполнение одного бизнес требования. Задачу можно разложить
на большое количество бизнес требований. Каждое бизнес требование
показывает как должна вести себя компьютерная программа в тех или
иных условиях. Фиксация (проверка) каждого бизнес требования при
помощи автоматического теста это как раз то к чему следует стремиться.

Если для всех бизнес требований будут написаны автоматические тесты,
то мы получим уникальную возможность быстро и с минимальными затратами
ресурсов проверять работу всей системы. Поверьте - это дорогого
стоит! К этому стремятся многие ИТ проекты и только единицы достигают
такого уровня автоматизации. Но это классный ориентир и отличная цель
к выполнению которой стоит стремиться.

Задача 1.14: разложите задачу на чёткие элементарные (как можно более
маленькие) бизнес требования. Сформулируйте (опишите) для каждого
бизнес требования конкретный тестовый сценарий (с входными данными и
ожидаемым результатом) для его проверки. Запрограммируйте
сформулированные вами сценарии в виде автоматических тестов.

PS: если это необходимо, то перед выполнением тестов инициализируйте
не один продакшн класс, а всю систему целиком. Это будут уже не
unit тесты, а acceptance тесты.
